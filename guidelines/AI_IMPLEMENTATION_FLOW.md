# AI Design System Implementation Flow (Template)

Цель: контролируемо перейти к machine-readable правилам (вариант 3) через управляемые этапы.

Важно:
- Этот файл задает процесс.
- Контент правил заполняет владелец системы (ты), не ассистент.
- Любой этап можно заморозить до подтверждения.

- Без двусмысленных формулировок.

Для каждого правила заполнить:

- scope:
- severity:
- conditions:

- must_not:

- evidence:

- [ ] Сформирован ruleset-файл по шаблону.

Gate:
- [ ] Нет правил без условий или без fallback.

## 4) Validator Prototype

Статус: [ ] not started [ ] in progress [ ] done

Что включить в v1:
- Проверка структуры spec.
- Проверка обязательных полей.
- Проверка привязки к rule ids.

Что НЕ включать в v1:
- Сложные эвристики.
- Полноценный AST/CSS parsing.

Артефакт:
- [ ] Прототип валидатора запускается локально.

Gate:
- [ ] Получаем предсказуемый отчет pass/fail.

Статус: [ ] not started [ ] in progress [ ] done

Артефакт:
- [ ] Один завершенный цикл spec -> validation -> review.

Gate:
- [ ] Владелец подтверждает готовность к масштабированию.
